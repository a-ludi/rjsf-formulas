# Publishing Design

**Date:** 2026-06-20
**Status:** Approved

## Overview

Publish `rjsf-formulas` as an open-source npm package on both the npm public registry and GitHub Packages, with CI, CD, a documentation site, and a live demo — all driven by GitHub Actions.

---

## 1. Repository Setup

Push the existing repo to GitHub. The access token in `.claude/github_token` is used for authenticated operations.

### `package.json` additions

GitHub Packages requires a scoped package name. Rename from `rjsf-formulas` to `@<owner>/rjsf-formulas` and publish under that name on both npm and GitHub Packages. `<owner>` is the GitHub username, resolved from `.claude/github_token` during implementation.

Add the following fields (existing fields are otherwise correct and unchanged):

```json
{
  "name": "@<owner>/rjsf-formulas",
  "description": "RJSF extension for computed fields driven by formulas",
  "author": "Arne Ludwig <arne.ludwig@posteo.de>",
  "license": "MIT",
  "repository": { "type": "git", "url": "https://github.com/<owner>/rjsf-formulas.git" },
  "homepage": "https://<owner>.github.io/rjsf-formulas/"
}
```

### New root files

- **`LICENSE`** — MIT license, year 2026, name Arne Ludwig
- **`README.md`** — project overview, installation, basic usage example, security section (see below), link to docs site and live demo
- **`CHANGELOG.md`** — initial file with `[Unreleased]` section and a `[0.1.0]` section (filled before tagging), following [Keep a Changelog](https://keepachangelog.com/en/1.0.0/)

### README security section

Warn explicitly that using `eval` as the evaluator with user-supplied formulas is unsafe (arbitrary code execution / XSS). Recommend sandboxed alternatives such as `expr-eval`, `mathjs`, or a Web Worker with a restricted evaluator.

## 2. CI Workflow (`.github/workflows/ci.yml`)

**Trigger:** push and pull requests to `main`

**Environment:** `ubuntu-latest`, Node 20

**Jobs (run in parallel):**

| Job | Command |
|-----|---------|
| `typecheck` | `tsc --noEmit` |
| `test` | `bun test` |

Intentionally minimal. No lint step until ESLint is configured.

---

## 3. Release Workflow (`.github/workflows/release.yml`)

**Trigger:** tag push matching `v*`

**Jobs (in dependency order):**

### 3a. `ci`
Re-runs typecheck and test on the tagged commit.

### 3b. `publish` (depends on `ci`)

**npm — via Trusted Publishers (OIDC):**
- No stored secret needed
- Requires `id-token: write` permission in the workflow
- One-time setup: register the workflow as a Trusted Publisher on npmjs.com
- Node setup with `registry-url: 'https://registry.npmjs.org'`
- Run `npm publish`

**GitHub Packages:**
- Uses the built-in `GITHUB_TOKEN` (no extra secret)
- Node setup with `registry-url: 'https://npm.pkg.github.com'`
- Run `npm publish`

**GitHub Release:**
- Created automatically from the tag
- Body populated from the `CHANGELOG.md` entry matching the tag version

### 3c. `deploy` (depends on `publish`)

Builds and deploys both the documentation site and the demo to GitHub Pages as a single combined artifact.

Steps:
1. Generate TypeDoc API reference → `website/api/` (markdown, via `typedoc-plugin-markdown`)
2. Build VitePress site (output: `website/.vitepress/dist/`)
3. Build Vite demo with `base: '/rjsf-formulas/demo/'`
4. Copy demo output into VitePress output at `dist/demo/`
5. Upload combined output as Pages artifact
6. Deploy via `actions/deploy-pages`

**Resulting URLs:**
- `https://<owner>.github.io/rjsf-formulas/` — documentation site
- `https://<owner>.github.io/rjsf-formulas/demo/` — live demo

**One-time repo settings:**
- GitHub Pages source set to **GitHub Actions** (not a branch)
- Pages permission: `pages: write`, `id-token: write` in the deploy job

## 4. Documentation Site (`website/`)

A VitePress project at `website/` (separate from `docs/` which holds internal tooling files).

### Site structure

| Section | Source |
|---------|--------|
| Quick Start | Hand-written markdown — installation, minimal usage example |
| Customization guide | Hand-written markdown — evaluator, schema keys, context modes, callbacks, `onFormulaError`, `onLoadingChange` |
| API Reference | Auto-generated by TypeDoc + `typedoc-plugin-markdown` from `src/` |

### TypeScript / JSDoc polishing

Before running TypeDoc, polish JSDoc comments on all public exports in `src/`:
- Every exported type, interface, and function gets `@param`, `@returns`, `@remarks`, and at least one `@example`
- No internal implementation details exposed; `@internal` tags hide private helpers

---

## 5. One-Time Manual Steps

Before the first release:

1. Push repo to GitHub using token in `.claude/github_token`
2. Register `.github/workflows/release.yml` as a Trusted Publisher on npmjs.com
3. In GitHub repo settings → Pages → set source to **GitHub Actions**

---

## 6. Secrets Summary

| Credential | How provided |
|------------|-------------|
| npm publish | OIDC (Trusted Publishers) — no secret stored |
| GitHub Packages publish | Built-in `GITHUB_TOKEN` |
| GitHub Pages deploy | Built-in `GITHUB_TOKEN` |
| GitHub API (repo creation) | `.claude/github_token` (local only, never committed) |
